5. Planning

A. What is Planning?

Planning is finding a sequence of actions (a plan) to achieve a goal. Unlike simple search, planning uses structured representations of states and actions.

B. Components

  • Initial State: Description of the world at start.
  • Goal State: Description of the world we want.
  • Operators (Actions):
    • Preconditions: What must be true to do the action.
    • Effects: What changes after the action (Add List / Delete List).

C. STRIPS (Goal Stack Planning)

STanford Research Institute Problem Solver.

Uses a stack to manage goals. It works backwards from the goal.

  1. Push Goal to Stack.
  2. If top is unsatisfied goal, push operator that satisfies it.
  3. If top is operator, check preconditions. If met, execute (update state). If not, push preconditions as new goals.

Limitation: Linear planning. Can struggle with interacting subgoals (Sussman Anomaly).

D. Non-Linear Planning (TWEAK)

Handles subgoals that interfere with each other. It uses Constraint Posting.

Instead of a strict sequence, it builds a partial ordering of steps and adds constraints (e.g., "Step A must come before Step B") to resolve conflicts.
